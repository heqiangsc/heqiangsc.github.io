ALTER TABLE 报1062 - Duplicate entry错误

Posted on 2021-12-02 In mysql

mysql

alter table

mysql版本：5.7.28

表结构

在添加字段的时候报错1062

问题原因：表数据较多在执行alter table期间有其他会话执行了造成duplicate entry的语句使online ddl在执行完结构变更回放缓存的时候报错。

Oline DDL的原理简单一点理解就是将DML操作缓存起来，等到DDL执行完成后重新应用缓存中的DML语句，如果在Oline DDL执行过程中，DML操作产生了Duplicate entry错误，并不会直接影响DDL操作，而是在DDL执行完成最终应用DML时报错，导致DDL执行失败。官方认为该问题是一种限制，并不是Bug，所以目前为止还没有得到解决。

经过测试只有在唯一键冲突的时候（insert,update）才会影响到DDL，主键冲突没有影响。

另外在执行insert into … on duplicate key…或者replace into语句的时候只要是更新了唯一键的字段也会影响online DDL的运行。

注意一点：
静态的执行 alter 是不会报错的，所以测试时很难发现问题，但是在生产环境时，就会因为报错而以为sql有问题，其实是Oline DDL 机制导致的，所以在mysql唯一键作为兜底的前提下，最好程序也要先查询一下来保证唯一性

读 禅

Posted on 2021-09-10 Edited on 2021-12-02 In 禅

禅

释迦牟尼的三句话

  1. 天上天下，唯我独尊。
  2. 奇哉，奇哉，一切众生皆有如来智慧德相，乃因妄想执着而不能证得。
  3. 以戒为师。
一、天上天下，唯我独尊

释迦牟尼佛为了度化人生，降生到人间，这是他到人间来的一个相。《妙法莲华经》云：「诸佛世尊，唯以一大事因缘故，出现于世。欲令众生开佛知见使得清净故，出现于世。」当时大地有六种吉祥震动，帝释丶梵天以各种化身示现，守护四周；天女散花，天众以天乐庄严，有龙王以殊胜美妙之甘露泉水沐浴佛身，有各种殊胜妙相与瑞兆产生。佛陀诞生后，随即于东丶西丶南丶北四方各走七步，每走一步，大地皆涌现宝莲承接佛足。佛陀一手指天，一手指地，然后说：「天上天下，唯我独尊 ；三界皆苦，吾当安之。」以此为佛陀于世间示现成佛之缘起，成为后世浴佛法会之由来。

释迦牟尼佛所说这个「我」，不单指释迦牟尼，也是指所有的人，包括你我他；这个「我」是指每个人心中独有的佛性，指一切众生的灵性。每一个人都有一个灵性，如果我们善用其心，就可以把心中的大智慧丶大慈悲统统体现出来。释迦牟尼佛告诉我们，每一个人都要相信自己。不管有钱没钱，不管是什么样的人，什么肤色，讲什么话，我们的灵性都是一样的，释迦牟尼有智慧，我们也有智慧，释迦牟尼可以成佛，我们也可以成佛。古时茶陵郁禅师悟道后说了一首诗偈：「我有明珠一颗，久被尘劳封锁，今朝尘尽光生，照破山河万朵。」所以在自性上，我们每个人是平等的，每个人都有灵性。

二、一切众生皆有如来智慧德相
1. 夜睹明星证道

释迦牟尼佛虽然生在皇宫，贵为王子，但是他经过对生活的观察，发现人有生老病死，不管你是什么人，有生就有老，有老就有病，病了以後就有死。社会上有好多不公平的事情。释迦牟尼佛为了寻求人生的真理，离开了皇宫荣华富贵的生活，出城去深山老林出家，释迦牟尼与五比丘在苦行林中修了六年苦行，忍受饥饿痛苦。35岁时，他意识到苦行无法达到解脱，光是修苦行还是不够，不能偏，不能太苦，也不能太乐，非苦非乐，才是真正修行之道。他到尼连禅河去沐浴，把多年苦行的垢秽洗净了。然後接受牧羊女的供养。坐于菩提树下，金刚座上，夜睹明星而悟道。依北传佛教说，释迦牟尼发愿不成正觉，永不起身。经过七天七夜，天正晓明，瞻望明星，悟得三明与四谛，证得无上正等正觉，而成为佛陀。

2. 揭示解开烦恼的密码

释迦牟尼佛开悟之后的心情跟我们凡人不一样，充满了法喜，悟到宇宙人生的真理，发现人生的秘密，他讲了一句话：「奇哉，奇哉，一切众生皆有如来智慧德相，乃因妄想执着而不能证得。」指出一切众生皆具佛性，皆可成佛。人人本具的佛性不生不灭，古今永存。

这段话出自《大方广佛华严经》卷第五十一，如来出现品第三十七之二：「复次佛子，如来智慧，无处不至，何以故？无一众生，而不具有如来智慧，但以妄想颠倒执着，而不证得，若离妄想，一切智丶自然智丶无碍智，则得现前。」

佛陀原本也是凡夫， 经过摸索，放下无明烦恼，彻悟本心而成就无上智慧 。在这里释迦牟尼佛说：「奇哉，奇哉。」奇哉是什么意思？就是太妙了，太妙了。哦！一切众生皆有如来智慧德性。在这以前不知道，通过修行，在菩提树下觉悟到宇宙人生的真理。这句话启示我们，不管是什么人，每个人都有如来藏妙如真心，都有佛性。释迦牟尼佛告诉我们，每个人都可以成佛。听到释迦牟尼这句话以后，我们就能够找到烦恼丶痛苦丶迷惑的根源，这是释迦牟尼佛在菩提树下开悟以后，解开烦恼的密码。

3.「妄想执着」是烦恼丶痛苦迷惑的根源

释迦牟尼佛开悟了，第一个发现宇宙人生最大的秘密，就是我们每一个人本来具有佛性，「一切众生皆有如来智慧德相」，为什么我们现在是凡夫，为什么我们不能成佛呢？「乃因妄想执着而不能证得」。正因为每一个众生本有这个佛性，所以我们可通过修行，通过发菩提心，行菩萨道，最后成佛。这就是一个理论依据，也就是说，我们现在看到佛前这个桃子，桃的种子最后结桃的果实，你看到这个香蕉，香蕉的种子到最后是香蕉的果实，这是一个道理。如是因，如是果，如果我们每个人没有佛性，你再怎么修你都没办法成佛。

我们烦恼从哪里来？痛苦从哪里来？迷惑从哪里来，就是从妄想执着而来。大家知道，当我们痛苦的时候，不是别人给我们痛苦，而是我们自己执着，自己妄想太多了。本来生活很开心快乐，但是去执着，去比较，人家有的我也要有，烦恼就是这样来的。

为什么释迦牟尼佛他成佛了，有智慧丶有神通丶有道德，而我们这些凡人却还有烦恼，还在六道里面轮回呢？既然大家都有佛性，现在为什么不能成佛？为什么我们没有智慧呢? 释迦牟尼佛告诉我们：「乃因妄想丶执着而不能证得。」我们凡人都会打妄想，现在这样想，等下那样想。在执着里面生活，执着一个人，执着一样东西，因为妄想执着，把我们本有佛性盖覆住了。虽然每一个人有灵性，有佛性，但是因为太执着，太计较了，清净心被盖覆住了，我们的灵性被束缚住了，佛性没办法显现出来，所以我们的烦恼就是从妄想执着而来。

三、以戒为师

释迦牟尼佛在人间生活到八十岁，就示现涅盘了，他就要离开人世，好多弟子都很着急，心里非常不安，释迦牟尼佛是我们的老师，他在世的时候，我们有问题可以去请教老师，老师不在怎么办？释迦牟尼佛答覆弟子说：要以戒为师。根据《遗教经》记载，释迦牟尼说：「汝等比丘！於我灭后，当尊重珍敬波罗提木叉（戒律）。如暗遇明，如贫人得宝。当知此则是汝等大师，若我住世，无异此也。」 佛说以后你们应该尊重戒律，如同黑夜遇到光明，如同贫穷的人得到宝藏。应该知道戒律是你们的大师，就如同我住世教导你们一样。所有出家人必须依照戒律修行，若不依戒律修行，佛法就会灭亡。若依照戒律修行，佛法则可以长久住世。

CentOS7使用firewalld打开关闭防火墙与端口

Posted on 2021-08-26 In linux

firewalld的基本使用

  • 启动： systemctl start firewalld
  • 关闭： systemctl stop firewalld
  • 查看状态： systemctl status firewalld
  • 开机禁用 ： systemctl disable firewalld
  • 开机启用 ： systemctl enable firewalld

systemctl是CentOS7的服务管理工具中主要的工具，它融合之前service和chkconfig的功能于一体。

  • 启动一个服务：systemctl start firewalld.service
  • 关闭一个服务：systemctl stop firewalld.service
  • 重启一个服务：systemctl restart firewalld.service
  • 显示一个服务的状态：systemctl status firewalld.service
  • 在开机时启用一个服务：systemctl enable firewalld.service
  • 在开机时禁用一个服务：systemctl disable firewalld.service
  • 查看服务是否开机启动：systemctl is-enabled firewalld.service
  • 查看已启动的服务列表：systemctl list-unit-files|grep enabled
  • 查看启动失败的服务列表：systemctl --failed

配置firewalld-cmd

  • 查看版本： firewall-cmd --version
  • 查看帮助： firewall-cmd --help
  • 显示状态： firewall-cmd --state
  • 查看所有打开的端口：firewall-cmd --zone=public --list-ports
  • 更新防火墙规则：firewall-cmd --reload
  • 查看区域信息: firewall-cmd --get-active-zones
  • 查看指定接口所属区域： firewall-cmd --get-zone-of-interface=eth0
  • 拒绝所有包：firewall-cmd --panic-on
  • 取消拒绝状态： firewall-cmd --panic-off
  • 查看是否拒绝： firewall-cmd --query-panic

那怎么开启一个端口呢

  • 添加
    firewall-cmd --zone=public --add-port=80/tcp --permanent （--permanent永久生效，没有此参数重启后失效）
  • 重新载入
    firewall-cmd --reload
  • 查看
    firewall-cmd --zone= public --query-port=80/tcp
  • 删除
    firewall-cmd --zone= public --remove-port=80/tcp --permanent
